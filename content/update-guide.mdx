<Steps>
  Newer versions of services often introduce changes that can break compatibility with our Coolify templates. 
  
  To ensure stability for all users, we pin each service to a specific version.
  
  If you’d like to update a service manually, follow the steps below:
  <Step>

  ### Find the Target Version
  
  Identify the version number you want to upgrade to.  
  For example, if your current Appwrite version is `v1` and you want to move to `v4`, note down the new version.

  </Step>

<Step>

### Review the Official Changelog

Go through the service’s official changelog to understand what has changed.  
Check for:  
- New or removed environment variables  
- Additional or deprecated services in the `docker-compose` file  
- Configuration changes required for new features

</Step>

<Step>

### Apply the Updates

Once you understand the required changes:  
- On **Coolify** and click **“Edit Compose”**  
  ![](/dev-samples/1.webp)
- Modify the compose file to include the new version and any necessary updates  
- Save your changes and **redeploy the app**

</Step>
</Steps>
